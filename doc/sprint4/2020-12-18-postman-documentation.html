General notes

Using Postman, calls to all the API endpoints, and their responses can be tested. In this document, we specify the API endpoints of the application and how they can be called to verify whether they work as expected or not.

API detail

Identity Service

The identity service takes care of authorization and authentication of the users.

Login (Student)

  • Curl
  • HTTP
curl -X POST -d '{
    "netid":"{{testStudentNetId}}",
    "password":"{{testStudentPassword}}"
}' "http://localhost:8083/login"
POST /login HTTP/1.1
Host: localhost:8083

    "password":"{{testStudentPassword}}"
}
